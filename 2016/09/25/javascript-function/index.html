JavaScript的函数

与其他语言的函数相比，ECMAScript的函数有一些特殊的灵活性。

1.不用指定返回类型

不返回类型的函数相当于返回undefined。

2.参数声明不是必要的

ECMAScript的函数参数就是一个数组arguments，不管有没有传参数或者传几个参数，少传的参数就是undefined。

3.没有重载

从上面可以知道，函数的参数不是必要的，所以ECMAScript没有函数签名的概念，决定函数的只有函数名。如果声明多个名称相同、参数不同的函数，浏览器只会取最后一个。

4.按值传递

ECMAScript中所有的函数参数都是按值传递，引用类型参数保存的是对象的地址，所以传递对象的地址后可以修改外部的对象。

5.没有块级作用域

ECMAScript中语句的花括号是没有作用域的，比如:

在java和c++中是有块级作用域的，ECMAScript这个特点容易引起困惑，不过我们可以使用函数表达式来模拟块级作用域。

6.可以改变作用对象

在ECMAScript中函数可以从一个对象传递到另一个对象，也可以直接使用call和apply，来改变函数的作用对象。call和apply区别在于call的第二个参数可以是任意类型，而apply的第二个参数必须是数组，即arguments。

7.this、arguments、callee、caller

这几个属性不是函数一开始就有的，是函数被调用时产生的。

this表示函数的调用者（callingContext），全局函数的this是window，上面person1.sayHi中sayHi的this是person1。this不是固定不变的，可以通过call、apply、bind和函数传递改变。

this不遵守作用域链查找规则。函数执行时，this 总是指向调用该函数的对象，规则可以简单概括如下：

  • 有对象就指向调用对象
  • 没调用对象就指向全局对象
  • 用new构造就指向新对象
  • 通过 apply 或 call 或 bind 可以改变this的指向

arguments表示调用函数时传递进来的参数列表，而不是函数声明时的参数列表。

caller表示调用这个函数的函数，如果调用这个函数的不是函数，则caller为null。

8.可以变身为构造函数

普通函数前面加一个关键字new，就变成构造函数了。一般构造函数的开头字母大写，以此来区分普通函数。

这其中可以分为几个步骤：

  1. 创建一个新的对象person1
  2. 将person1的constructor设为Person
  3. Person.call(person1, “tom”, 20)
  4. 返回person1

函数有个prototype属性，表示这个函数构造出来的对象的原型，所谓的原型就是一个对象保存一些共享的属性和方法。
